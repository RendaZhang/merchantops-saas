# Security Policy

## Supported Release Line

MerchantOps SaaS is currently published as an alpha preview.

Current support posture:

- `v0.2.x-alpha`: best effort
- older preview milestones: no active support guarantee

## Reporting A Vulnerability

Please do not disclose security-sensitive details in a public GitHub issue.

Use one of these private channels instead:

- Email: `rendazhang@qq.com`
- GitHub private vulnerability reporting, if available for this repository

When reporting a vulnerability, please include:

- the affected version or commit
- a short description of the issue
- reproduction steps or a proof of concept when safe to share privately
- impact assessment, if known

## Response Expectations

- Initial triage is handled on a best-effort basis
- This repository is currently an alpha-stage open-source project, not a production support offering
- Fix timing depends on severity, maintainers' availability, and the maturity of the affected release line

## Public Disclosure

Please wait for a maintainer response before any public disclosure.

If the issue is confirmed, the preferred order is:

1. private report
2. maintainer triage
3. coordinated fix
4. release note or advisory after remediation is available
